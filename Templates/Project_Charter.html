Project Objective

The purpose of this project is to create an intuitive, interactive dashboard/interface using Flask that incorporates key project management elements. These include a project charter, work breakdown structure (WBS), statements of work (SOW), Gantt chart with a working calendar, project network diagram, critical path analysis, cost estimates, team-building exercises, and templates for status/progress reports and risk management plans. The goal is to support effective project management through accessible and user-friendly web-based tools.

Project Scope

  • Deliverables: Develop a Flask-based web application that includes:
    • Project charter
    • Work breakdown structure
    • Statements of work
    • Gantt chart and working calendar
    • Project network diagram
    • Critical path analysis
    • Cost estimation module
    • Team-building exercises
    • Status and progress report templates
    • Risk management plan
  • User Requirements: Ensure the dashboard is user-friendly and designed to support team collaboration and communication effectively.

Summary Schedule

The project must be completed and turned in by 12/05/2024.

Summary Budget

Budget is limited, primarily covering essential tools and resources. Open-source tools and technologies, including Flask, will be used to minimize costs. Any additional expenses will require authorization by the project sponsor.

Project Success Criteria

  • Completion of all listed project deliverables within the specified timeline.
  • Functionality of key project management features, including the ability to create, modify, and share Gantt charts, WBS, and project network diagrams.

Assumptions and Constraints

  • Assumptions: Team members will dedicate adequate time for their respective roles. Flask will be the primary web development framework.
  • Constraints: Budget for tools and resources is limited. All deliverables must be presented in a web-based format accessible to all stakeholders.

  • Jacob Dorow & Cooper Johnson (Researchers): Research best practices, frameworks, and suitable modules to use.
  • Mark Dsilva, Mitch Lovdal, Noah Nguyen (Designers): Create the user interface, wireframes, and design elements.
  • Nolan Worley, Matt Morris, Dilpreet Supra, Ian Listopad, Raymond Ha (Developers): Develop the Flask application, integrate modules, and perform testing.

Communication Standards

  • Team Meeting: Weekly meetings will be conducted during class time to make progress on the project.
  • Online Communication: Asynchronous communication to discuss strategy and coordinate between team members will be conducted through Microsoft Teams.
